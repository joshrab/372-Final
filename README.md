# DGI — Duke Gardens Index


## Project Goal
The goal of this project was to get Duke students more engaged with the wonderful Duke Gardens by identifying and learning more about individual plant species native to the Duke Gardens. I hope to turn it into an app with data persistence that allows users to keep track of the plants they have found--hence the name, Duke Gardens 'Index'!


## What it Does

DGI is a plant identification and information system built around Duke University's Sarah P. Duke Gardens. A user uploads a photo of a plant, and the system identifies the species using a fine-tuned ResNet18 image classifier trained on species native to Duke Gardens. Once identified, the user can ask natural language questions about the plant, which are answered using a retrieval-augmented generation (RAG) pipeline backed by a Wikipedia-sourced knowledge base. On GPU, answers are generated by Phi-2, a local transformer language model. On CPU, the system returns the full knowledge base entry for the identified species. The result is a cohesive, end-to-end system that connects computer vision, semantic retrieval, and language generation around a single real-world botanical domain.

## Evaluation

The ResNet18 classifier was trained on a dataset of 102 plant species sourced from the Duke Gardens species list via iNaturalist, with approximately 250 images per species (~22,000 images total). The model was trained with a 70/15/15 train/validation/test split.

| Metric | Value |
|---|---|
| Test Accuracy | 65.7% |
| Number of Classes | ~102 |
| Training Images | ~22,000 |
| Backbone | ResNet18 (fine-tuned) |

RAG retrieval uses cosine similarity over `sentence-transformers/all-MiniLM-L6-v2` embeddings against a Wikipedia knowledge base covering all species in the dataset. Retrieval is anchored to the classified species via exact name matching before falling back to semantic search.
