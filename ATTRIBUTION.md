# Attribution

This project was developed with assistance from ChatGPT and Claude (Anthropic). AI was used for implementation support (API usage, data processing, training loops), debugging, RAG pipeline construction (embeddings and retrieval), frontend development, and documentation generation (README, SETUP, and ATTRIBUTION). All AI-generated code was reviewed, modified, and understood by the author.

AI contributions were not used verbatim in all cases, or rejected in others. In several instances, generated code required modification, debugging, or restructuring to align with the project requirements and existing codebase. Notably:

* The iNaturalist API validation logic required reworking to correctly verify species names. This involved testing across both local and Colab environments, handling API inconsistencies, and consolidating valid species into a stable list used throughout the project.
* AI-generated suggestions for integrating external libraries (e.g., transformers for Phi-2) required consulting official documentation to resolve version mismatches, deprecated arguments, and loading issues.
* Some AI-generated code conflicted with existing project structure or homework-based code; these suggestions were selectively adapted or rejected to preserve consistency with prior work and ensure correctness.
* Debugging steps included fixing JSON formatting issues, resolving dataset mismatches, and ensuring compatibility between locally generated artifacts and Colab execution.
* Project structure and file paths were incorrectly generated by AI in several instances, requiring manual intervention to correct import paths, output directories, and cross-environment file references.
* The Flask frontend (drag-and-drop UI, chat interface) was generated by Claude and required manual edits including renaming the project to "DGI — Duke Gardens Index", adjusting color scheme and typography to reflect Duke's visual identity, and modifying layout and styling details.
* The CPU fallback behavior in the RAG answer function was explicitly added by the author — AI would have returned an empty or overconfident response on CPU. The author instructed the system to instead return the full knowledge base entry with a clear disclaimer that generative answers require GPU, making the system honest and functional in both environments.
* The modular src/ file structure (`app.py`, `classifier.py`, `rag.py`, `main.py`, `make_class_names.py`) was scaffolded with AI assistance but required significant manual debugging of inter-module imports, incorrect method signatures, and broken references between files.

The overall structure of the notebooks follows the general format of course homework assignments, adapted for this project. The Transfer Learning homework informed the use of ResNet18, fine-tuning strategy, and training pipeline, while the RAG homework informed the embedding and retrieval workflow.

Some code was adapted from course materials by Brandon Fain, including training utilities such as `plot_training_curves`. Training and testing methods were pulled from prior homework submissions by Joshua Rabbani and adapted for the purposes of the project.
